# UDo_Software_Engineering
This project is a software design for a restaurant management system.
The main goal was design an efficient application that automates restaurant operations.
The system includes features to process customer orders, manage reservations and handle staff and inventory operations.
The system stores booking details such as customer names, table numbers, dates and reservation status.
It tracks customer orders by monitoring menu items, quantities and special instructions.
The software monitors stock quantities and calculates reorder levels for timely procurement.
It maintains employee records regarding shift schedules and total hours worked.
The team used the Model View Controller (MVC) design pattern to organize the code and separate concerns.
We followed naming conventions, using PascalCase for class names and camelCase for methods.
The implementation used OOP principles like encapsulation and abstraction to keep the code clean.
We used GitHub as a version control system to track changes and collaborate smoothly. 
The project followed a systematic process, including requirements analysis, system design and implementation.
The team worked successfully by maintaining open communciation and meeting all project objectives effectively.
The experience helped me sharpen my analytical, collaboration and project management skills.
